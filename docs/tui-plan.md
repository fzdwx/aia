# tui plan

## 目标

终端界面要先做到三件事：

1. 不闪烁
2. 可恢复会话
3. 能显示代理事件，而不只是聊天文本

## 渲染原则

- 使用单独的终端屏幕缓冲区，避免污染原始终端内容
- 采用稳定的帧刷新，而不是每次整屏清空后重画
- 后续真正接入终端库时，优先选择差量绘制而不是手写整屏擦除
- 工具调用、状态变化、交接标记都应进入统一事件流，再由界面订阅展示

## 第一阶段界面范围

- 左侧：会话与阶段信息
- 中间：消息与事件时间线
- 底部：输入框与当前模型信息
- 右侧：工具活动与交接摘要

## 当前已落地的过渡交互

- 程序启动时已具备最小终端内 provider 创建与选择流程
- 后续真正 TUI 需要复用这条 provider 管理链路，而不是重新做一套配置入口
- 当前 loop 已通过统一事件方法取回事件；后续真正 TUI 应作为事件订阅者渲染，而不是直接读取散落输出
- 当前最小 TUI 已区分“回放面板”和“本次运行”两个区域
- 当前最小 TUI 已具备焦点切换与快捷键提示（Tab / ↑↓ / F2）

## 明确延后

- 动画细节打磨
- 多窗口桌面联动
- 异步子代理的复杂可视化

先把共享运行时跑稳，再把界面做成这个运行时的观察面板。
