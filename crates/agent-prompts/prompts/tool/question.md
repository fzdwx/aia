Use this tool when you need the user to answer a structured clarification question before you can continue.

Only use it when the current session supports interactive components.

Rules:
- Ask only when there is a real ambiguity or decision point.
- Prefer concise, clear questions.
- When presenting choices, make the options meaningfully different.
- You may include a recommended option and a short reason, but never assume the recommendation is accepted.
- Treat the result as structured data. Check its `status` before proceeding.
- If the result is `cancelled`, `dismissed`, `timed_out`, or `unavailable`, do not act as if the user approved anything.
