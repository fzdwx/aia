## Design Context

### Users
本项目面向两类核心用户：一类是希望直接使用 agent 完成日常工作的 C 端用户，另一类是需要驱动 coding agent、开发软件、排查运行链路的程序员。

他们通常处于高专注、任务导向的工作场景中，希望在一个统一工作台里完成对话驱动、工具调用观察、trace 检查、provider/channel 管理与 runtime 调试，而不是在多个低效页面之间来回切换。

### Brand Personality
品牌气质应保持：好用、流畅、审美现代。

界面语气应让用户感到速度、现代感、愉悦和精准。它不是炫技型产品，也不是夸张的营销式界面，而是一种让人愿意长时间停留、并且能高效处理复杂任务的专业工作台。

### Aesthetic Direction
当前代码库已经形成了一个可延续的方向：工程工作台、简约黑白基底、清晰的信息层次、较克制的动效、带少量质感的半透明面板与网格/颗粒背景。

后续视觉方向应继续围绕高密度但不压迫的专业工作台展开。主色以黑白灰为骨架，高亮色可使用接近 Vue 主题的绿色作为关键状态、强调动作和少量数据点缀。避免大面积空白、低信息密度、重复模块堆砌，以及“看起来很干净但效率很低”的展示型布局。

不追求某个现成参考产品的模仿，但应明确避免：稀疏到发空的页面、大块重复卡片、为了极简而牺牲信息承载和操作效率的设计。

### Design Principles
1. 一站式优先：对话、工具、trace、配置、调试应被组织成统一工作台，而不是分裂成多个互相割裂的表层页面。
2. 高信息密度，但保持清晰节奏：减少无意义留白和重复块，用层级、分组、排版与对比建立可扫描性，而不是靠摊大面积空间。
3. 速度感必须可见：交互反馈、状态切换、流式更新和布局稳定性要让用户感到快、顺、不中断，避免闪烁和迟滞感。
4. 视觉基调保持现代克制：以黑白灰为主，少量 Vue 绿色做强调；避免杂色、廉价装饰感和过度拟物/营销化表达。
5. 为长时间使用而设计：优先保证阅读舒适、操作连续性和复杂任务可管理性；在可能的前提下兼顾主题、可访问性和更广泛用户的使用需求。
